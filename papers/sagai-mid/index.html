SAGAI-MID: A Generative AI-Driven Middleware for Dynamic Runtime Interoperability

Authors: Oliver Aleksander Larsen, Mahyar T. Moghaddam

Venue: SAGAI 2026 Workshop at IEEE ICSA 2026

Status: Accepted — to appear in proceedings after the conference (June 22–26, 2026, Amsterdam)

Abstract

We present SAGAI-MID, a FastAPI-based middleware that uses large language models to dynamically resolve schema mismatches at runtime. The system implements a five-layer pipeline with hybrid detection, dual resolution strategies (DIRECT transformation vs. CODEGEN), and a three-tier safeguard stack. Evaluated across 10 interoperability scenarios using 6 different LLMs, the best configuration achieves 0.90 pass@1 accuracy, demonstrating practical viability of LLM-driven runtime schema adaptation.

TL;DR

A middleware that uses LLMs to automatically fix API schema mismatches at runtime — achieving 90% accuracy across 10 test scenarios and 6 different language models.

Frequently Asked Questions

How does SAGAI-MID resolve schema mismatches?

It uses a five-layer pipeline: schema extraction, hybrid mismatch detection, LLM-based resolution (via direct transformation or code generation), and a three-tier safeguard stack for validation.

What accuracy does SAGAI-MID achieve?

The best LLM configuration achieves 0.90 pass@1 accuracy across 10 diverse interoperability scenarios, with the CODEGEN strategy generally outperforming DIRECT transformation.
